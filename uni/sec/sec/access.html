Access Control and Integrity[..] [top]

1Definitions [2.1][up]

Access control implements authorization, granting or denying access accordingly.

Subject

an entity that may access objects

Object

system resources

Modes/permissions

ways in which objects can be accessed

Authorized accesses

described by a set of tuples

we typically distinguish between permissions which can be granted and permissions which have been granted

Discretionary access control (DAC)

each object has a subject as owner, which controls its permissions (e.g. Unix file permissions)

Mandatory access control (MAC)

administrator sets all permissions with central control, preventing badly behaving users affecting security (e.g. SELinux)

3Security models[up]

3.1Bell-LaPadula model [2.2][up]

Aims to achieve confidentiality only.

  • Permissions

  • Set of security levels comparable by (should be a lattice)

  • Maximum security level for each subject (clearance)

  • Current security level for each subject

  • Classification for each object

  • Set of allowable permissions

  • Set of granted permissions

Systems evolve through states, which are individually considered secure if they satisfy the conditions below, and the system is secure if every implementation action preserves these conditions.

Note that the permission write includes permissions to read the file (something like rw mode), while append does not.

3.2Role-based access control (RBAC)[up]

  • Privileges are rights to execute specific operations (usually OS functions), and act as an intermediate layer between users and operations.

  • Roles are groups with some privileges, which users are assigned to.
